# 要件定義書: Tab Switcher

## 概要

VSCodeのタブ切り替え（Ctrl+Tab）に着想を得た、MRU（Most Recently Used）順のタブスイッチャー Chrome拡張機能。キーボード操作でブラウザのタブを素早く切り替えることを目的とする。

## 要件

### 要件 1: オーバーレイの起動

**ユーザーストーリー:** ブラウザユーザーとして、キーボードショートカットでタブ一覧を表示したい。素早くタブを見つけて切り替えるため。

#### 受け入れ基準

1. ユーザーが `Cmd+Shift+Space`（Mac）/ `Ctrl+Shift+Space`（Windows/Linux）を押したとき、システムは現在のページ上にオーバーレイを表示すること
2. 修飾キーを離したとき、オーバーレイを閉じること（キー押し続けモード時）
3. `Escape` を押したとき、オーバーレイを閉じること（通常モード時）
4. ショートカットが他の拡張機能と競合する場合、ユーザーは `chrome://extensions/shortcuts` からカスタマイズできること

### 要件 2: MRU順タブリスト表示

**ユーザーストーリー:** ブラウザユーザーとして、最近使ったタブ順にリストを見たい。直前に使っていたタブにすぐ戻れるようにするため。

#### 受け入れ基準

1. オーバーレイが表示されたとき、タブリストはMRU（最近アクセスした順）で表示されること
2. タブリストの各項目はファビコン、タイトル、URLを表示すること
3. ユーザーがタブを切り替えたとき、そのタブのアクセス時刻が更新され、次回のMRU順序に反映されること
4. タブリストに表示されるのは、オーバーレイを起動したウィンドウ内のタブのみであること（別ウィンドウのタブは候補に含めない）

### 要件 3: キー押し続けモード（VSCode風）

**ユーザーストーリー:** ブラウザユーザーとして、VSCodeのCtrl+Tabのように修飾キーを押し続けている間だけリストを表示し、離すと確定させたい。最小限の操作でタブを切り替えるため。

#### 受け入れ基準

1. ユーザーが `Cmd+Shift`（Mac）/ `Ctrl+Shift`（Windows/Linux）を押し続けている間、オーバーレイは表示され続けること
2. オーバーレイ表示中に `Space` を押したとき、フォーカスが次のタブに移動すること
3. 修飾キー（`Cmd+Shift` / `Ctrl+Shift`）を離したとき、フォーカスされているタブに切り替わり、オーバーレイが閉じること

### 要件 4: 通常モード操作

**ユーザーストーリー:** ブラウザユーザーとして、矢印キーやEnterキーでもタブを選択・切り替えしたい。じっくりタブを選びたい場面があるため。

#### 受け入れ基準

1. オーバーレイ表示中に `↓` キーを押したとき、フォーカスが次のタブに移動すること
2. オーバーレイ表示中に `↑` キーを押したとき、フォーカスが前のタブに移動すること
3. オーバーレイ表示中に `Enter` を押したとき、フォーカスされているタブに切り替わり、オーバーレイが閉じること
4. フォーカスがリストの末尾にある状態で `↓` を押したとき、フォーカスはリストの先頭に戻ること

### 要件 5: タブを閉じる

**ユーザーストーリー:** ブラウザユーザーとして、オーバーレイからタブを閉じたい。不要なタブを画面を離れずに整理するため。

#### 受け入れ基準

1. 各タブ項目にバツボタン（×）が表示されていること
2. バツボタンをクリックしたとき、そのタブが閉じられること
3. タブを閉じたとき、リストから即座に消え、フォーカスは隣のタブに移動すること
4. 最後の1つのタブは閉じられないこと

### 要件 6: ファジー検索

**ユーザーストーリー:** ブラウザユーザーとして、テキスト入力でタブを検索したい。大量のタブから目的のものを素早く見つけるため。

#### 受け入れ基準

1. `Cmd+Shift+P`（Mac）/ `Ctrl+Shift+P`（Windows/Linux）でファジー検索モードのオーバーレイが開くこと
2. テキストを入力したとき、タブリストがタイトルとURLに対してファジーマッチで絞り込まれること
3. マッチした部分がハイライト表示されること
4. 検索結果が0件の場合、「該当なし」のメッセージを表示すること
5. 検索テキストをすべて削除したとき、全タブのリストに戻ること
6. `Enter` を押したとき、フォーカスされているタブに切り替わること
7. `Escape` を押したとき、オーバーレイを閉じること

## 非機能要件

### コードアーキテクチャ
- **単一責任原則**: Background（MRU管理）、Content Script（UI）、メッセージングを明確に分離
- **モジュラー設計**: タブ管理ロジック、UIコンポーネント、キーボードハンドリングを独立したモジュールに
- **明確なインターフェース**: Background ↔ Content Script 間のメッセージ型を厳密に定義

### パフォーマンス
- オーバーレイの表示は 100ms 以内
- タブリストの描画は 50 タブまでスムーズに動作すること
- ファジー検索の絞り込みは入力に対してリアルタイム（体感遅延なし）

### ユーザビリティ
- キーボードのみで全操作が完結すること
- ダークテーマのオーバーレイで、どのWebページ上でも視認性を確保すること

## スコープ外（v1）

- スクリーンショットプレビュー
- マウスによるドラッグ&ドロップでのタブ並び替え
- タブのピン留め・グループ化操作
- Firefox対応（Chrome のみ）
