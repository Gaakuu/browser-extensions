# 要件定義書

## 概要

Chrome 拡張機能「Screenshot」は、ブラウザで閲覧中のWebページのスクリーンショットを手軽に撮影・保存するツール。ショートカットキー（Cmd+Shift+2）で起動すると画面全体が暗転し、要素検知モードに入る。ホバーで要素をハイライト → クリックで撮影、またはドラッグでトリミングモードに切り替えて任意範囲を撮影できる。ツールバーからページ全体・表示領域のキャプチャも選択可能。撮影後はクリップボードへのコピーまたはファイル保存ができる。

## 要件

### 要件 1: 撮影モードの起動とツールバー

**ユーザーストーリー:** ユーザーとして、ショートカットキーですぐに撮影モードを起動し、ツールバーから撮影方法を選びたい。なぜなら、最小限の操作でキャプチャを開始したいから。

#### 受け入れ基準

1. Cmd+Shift+2（Mac）/ Ctrl+Shift+2（Windows/Linux）で撮影モードが起動すること
2. 起動すると画面全体が暗転オーバーレイで覆われること
3. 画面の上部または下部にツールバーが表示されること
4. ツールバーには「全画面」「表示領域」「設定（歯車）」のボタンがあること
5. Escキーで撮影モードを終了（暗転解除 + ツールバー非表示）できること

### 要件 2: 要素検知モード（デフォルト）

**ユーザーストーリー:** ユーザーとして、マウスを動かすだけでDOM要素を自動検知し、クリックでその要素をキャプチャしたい。なぜなら、特定のUI部品を正確に素早く切り取りたいから。

#### 受け入れ基準

1. 撮影モード起動時、デフォルトで要素検知モードになること
2. マウスホバーした位置のDOM要素が検知され、その要素の領域だけ暗転が解除される（明るく表示される）こと
3. ハイライトはマウス移動に追従してリアルタイムに切り替わること
4. クリックで要素を確定し、その要素の矩形範囲がキャプチャされること
5. 要素の周囲にパディング（デフォルト: 8px）を含めてキャプチャすること

### 要件 3: トリミングモード（ドラッグ切り替え）

**ユーザーストーリー:** ユーザーとして、要素検知モード中にドラッグ操作で任意の矩形範囲を選択してキャプチャしたい。なぜなら、要素単位ではなく自由な範囲で切り取りたい場合があるから。

#### 受け入れ基準

1. 要素検知モード中にマウスを押し込んでドラッグを開始すると、トリミングモードに切り替わること
2. ドラッグ中、選択範囲が暗転解除（明るく表示）されること
3. ドラッグ中、選択範囲のサイズ（px）が表示されること
4. マウスボタンを離すと、選択範囲がキャプチャされること
5. Escキーでトリミングをキャンセルし、要素検知モードに戻れること

### 要件 4: ページ全体のキャプチャ

**ユーザーストーリー:** ユーザーとして、スクロールが必要な長いページ全体を1枚のスクリーンショットとして撮影したい。なぜなら、ページ全体の内容を一度に記録・共有したいから。

#### 受け入れ基準

1. ツールバーの「全画面」ボタンをクリックしたとき、ページの先頭から末尾までがスクロールキャプチャされ、1枚の画像に結合されること
2. キャプチャ中、プログレス表示（撮影進行状況）が表示されること
3. 非常に長いページの場合、上限（例: 10,000px）を設けて警告を表示すること

### 要件 5: 表示領域のキャプチャ

**ユーザーストーリー:** ユーザーとして、現在ブラウザに表示されている領域をそのままスクリーンショットとして撮影したい。なぜなら、見ている画面をすぐに共有したいから。

#### 受け入れ基準

1. ツールバーの「表示領域」ボタンをクリックしたとき、現在の表示領域が画像としてキャプチャされること
2. キャプチャ後、プレビュー画面が表示されること

### 要件 6: クリップボードへの自動コピー

**ユーザーストーリー:** ユーザーとして、撮影したスクリーンショットが自動的にクリップボードにコピーされてほしい。なぜなら、撮影後すぐに Slack やドキュメントに貼り付けたいから。

#### 受け入れ基準

1. キャプチャが完了した時点で、画像が自動的にクリップボードにコピーされること
2. コピー完了後、成功の通知（トースト）が表示されること
3. コピーに失敗した場合、エラーメッセージが表示されること

### 要件 7: ファイル保存

**ユーザーストーリー:** ユーザーとして、撮影したスクリーンショットをPNGファイルとしてダウンロード保存したい。なぜなら、ローカルにファイルとして残しておきたいから。

#### 受け入れ基準

1. プレビュー画面で「保存」ボタンをクリックしたとき、PNG形式でダウンロードされること
2. ファイル名は `screenshot_YYYY-MM-DD_HH-MM-SS.png` の形式であること
3. Chrome のデフォルトダウンロードフォルダに保存されること

## 非機能要件

### コードアーキテクチャ
- **単一責任原則**: 各ファイルは明確な1つの責務を持つ
- **モジュラー設計**: キャプチャモード、UI、保存処理を分離し再利用可能にする
- **明確なインターフェース**: Background ↔ Content Script 間のメッセージ型を厳密に定義する
- **既存パターンの踏襲**: `extensions/tab-switcher/` のエントリポイント構成、Shadow DOM パターン、メッセージングパターンに従う

### パフォーマンス
- 表示領域のキャプチャは 500ms 以内に完了すること
- ページ全体のキャプチャ中、UIがフリーズしないこと（非同期処理）
- 要素検知のホバー追従は 16ms（60fps）以内に更新されること
- キャプチャ画像の品質はデバイスピクセル比を考慮すること

### セキュリティ
- 必要最小限の Chrome 権限のみを要求する（`activeTab`, `clipboardWrite`, `downloads`）
- Content Script はホストページの DOM を読み取るが、機密データの送信は行わない

### ユーザビリティ
- 撮影モード起動後、操作が直感的に理解できること（暗転 + ホバーハイライトで自然に誘導）
- キャプチャからコピー/保存まで最短2クリックで完了できること
- ダークモード / ライトモードの両方に対応すること（`packages/ui/` のテーマを利用）

## スコープ外

- 画像への注釈・矢印・テキストの書き込み機能（v2 で検討）
- 動画 / GIF の録画機能
- 外部サービス（Slack, Google Drive 等）への直接アップロード
- 撮影履歴の管理・ギャラリー機能
- JPEG や WebP など PNG 以外の出力形式
